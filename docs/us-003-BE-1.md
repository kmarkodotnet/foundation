# us-003-BE-1 | BE: RBAC Authorization + Policy Setup + AuthorizationBehaviour

**Story:** US-003 | [Auth] Jogosultság megakadályozása  
**Layer:** Backend  
**Priority:** Magas  
**Size estimate:** M  
**Sprint:** Sprint 1  
**Depends on:** us-001-BE-1.md  
**Blocks:** minden egyéb BE task (RBAC minden endpointon szükséges)

---

## Story

> As a **rendszer**, I want to prevent unauthorized access, so that every user can only access data and functions corresponding to their role.

---

## Scope

Implements the RBAC policy registration, `RequireRoleAttribute`, `AuthorizationBehaviour` in the MediatR pipeline (including locked-application guard), and exception middleware mappings. Does NOT implement specific endpoint permissions — those are part of individual feature tasks.

---

## Implementation Checklist

### Domain Layer
- [ ] `ForbiddenException` in `Domain/Exceptions/ForbiddenException.cs`
- [ ] `NotFoundException` in `Domain/Exceptions/NotFoundException.cs`

### Application Layer
- [ ] `Application/Common/Behaviours/ValidationBehaviour.cs` — FluentValidation pipeline (generic `IPipelineBehavior<TRequest, TResponse>`)
- [ ] `Application/Common/Behaviours/AuthorizationBehaviour.cs`:
  - Reads `[RequireRole]` attribute from request type
  - Checks `ICurrentUserService.Role` against allowed roles
  - Checks locked-application guard: if request implements `IApplicationCommand` and Application.IsLocked() and user is not Admin → throws `ForbiddenException`
- [ ] `Application/Common/Behaviours/AuditBehaviour.cs` — auto-audit for `ICommand` implementations
- [ ] `Application/Common/Interfaces/ICurrentUserService.cs` — Guid UserId, string Email, UserRole Role, bool IsAdmin()
- [ ] `Application/Common/Interfaces/IApplicationCommand.cs` — marker interface with Guid ApplicationId

### Infrastructure Layer
- [ ] `Infrastructure/Services/CurrentUserService.cs` — implements ICurrentUserService; reads claims from IHttpContextAccessor
- [ ] DI registration in `Infrastructure/DependencyInjection.cs`

### API Layer
- [ ] `API/Attributes/RequireRoleAttribute.cs` — custom `[RequireRole(UserRole.Admin, UserRole.PalyazatiMunkatars)]` attribute
- [ ] Policy registration in `API/Program.cs`:
  ```
  CanCreateApplication = Admin + PalyazatiMunkatars
  CanApproveApplication = Admin + Elnok
  CanManageInvoices = Admin + Penzugyes
  CanManageUsers = Admin
  CanViewAuditLog = Admin + Elnok
  CanManageCodelists = Admin
  CanManageGranters = Admin + PalyazatiMunkatars
  CanManageVendors = Admin + PalyazatiMunkatars + Penzugyes
  ```
- [ ] `API/Middleware/ExceptionMiddleware.cs`:
  - `ForbiddenException` → 403
  - `NotFoundException` → 404
  - `ValidationException` (FluentValidation) → 400 RFC 7807
  - Unhandled exception → 500
- [ ] Global `[Authorize]` on all controllers (anonymous only for `/auth/google-callback`)

### Authorization
- [ ] All non-auth endpoints require valid JWT
- [ ] Role-based checks via `AuthorizationBehaviour`

## API Contract

No new endpoints — this task sets up cross-cutting concerns.

**Error response format (RFC 7807):**
```json
{
  "type": "https://palyazat.alapitvany.hu/errors/forbidden",
  "title": "Nincs jogosultságod ehhez a művelethez.",
  "status": 403,
  "traceId": "00-abc123..."
}
```

---

## FluentValidation Rules
Implemented via `ValidationBehaviour` — applies to all commands with registered validators.

---

## MediatR Pipeline Notes
- Order: ValidationBehaviour → AuthorizationBehaviour → AuditBehaviour → Handler
- `AuthorizationBehaviour` also handles locked-application guard via `IApplicationCommand`
