## Story

> As a **rendszer**, I want to prevent unauthorized access, so that every user can only access data and functions corresponding to their role.

---

## Scope

`RoleGuard` for route-level protection, `HasRoleDirective` for element-level visibility, global `ErrorInterceptor` for 403/500 handling. Authentication guard was done in us-001-FE-1.

---

## Implementation Checklist

### Guards
- [ ] `src/app/core/auth/role.guard.ts` — `canActivate`: reads `data.roles` from route; calls `authService.hasAnyRole(roles)`; if false → redirect `/403` (nem `/login` — a 403 és az "unauthenticated" különböző eset; AuthGuard kezeli a `/login` redirect-et)
- [ ] `RoleGuard` applied to admin routes in `app-routing.module.ts`: `canActivate: [AuthGuard, RoleGuard], data: { roles: ['Admin'] }`
- [ ] `roles.constants.ts` létrehozása: `export const ROLES = { ADMIN: 'Admin', ELNOK: 'Elnok', PALYAZATI_MUNKATARS: 'PalyazatiMunkatars', PENZUGYES: 'Penzugyes', MEGTEKINTO: 'Megtekinto' } as const;` — minden `*hasRole` direktíva és guard erre hivatkozzon, ne string literálra (ékezet nélküli formák a kanonikusak, ezek szerepelnek a JWT claim-ben)

### Directives
- [ ] `src/app/shared/directives/has-role.directive.ts` (full implementation):
  ```typescript
  @Directive({ selector: '[hasRole]' })
  export class HasRoleDirective {
    @Input() set hasRole(roles: string[]) {
      this.viewContainer.clear();
      if (this.authService.hasAnyRole(roles)) {
        this.viewContainer.createEmbeddedView(this.templateRef);
      }
    }
  }
  ```
- [ ] Export from `SharedModule`
- [ ] Usage example: `<button *hasRole="['Admin', 'PalyazatiMunkatars']">Szerkesztés</button>`

### HTTP Interceptor
- [ ] `src/app/core/http/error.interceptor.ts`:
  - 403 → MatSnackBar toast: „Nincs jogosultságod ehhez a művelethez."
  - 500 → MatSnackBar toast: „Szerverhiba történt. Kérjük, próbálja újra."
  - 401 → delegál AuthInterceptorhoz (already handled)
- [ ] Register in `CoreModule` providers

### Navigation / Routing
- [ ] `src/app/features/error/forbidden/forbidden.component.ts` — 403 oldal (kötelező; a RoleGuard ide redirectel)
- [ ] Route: `{ path: '403', component: ForbiddenComponent }` in `app-routing.module.ts`
