## Story

> As a **pályázati munkatárs**, I want to edit the call data so that I can fix typos or update grant conditions.

---

## Scope

Edit form for the application call step, pre-populated with existing data. Available when step is not LOCKED.

---

## Implementation Checklist

### Feature Module
- [ ] `detail/workflow/step-call/step-call.component.ts` — **us-012-FE-1 hozza létre** (read-only nézet); ez a task **bővíti** szerkesztési móddal:
  - „Szerkesztés" gomb (hasRole Admin/PalyazatiMunkatars, LOCKED esetén disabled)
  - Szerkesztési mód: inline form vagy modal

### Service
- [ ] `ApplicationsService.updateApplication(id: string, cmd: UpdateApplicationRequest): Observable<ApplicationDto>` — PUT /api/v1/applications/:id
- [ ] `UpdateApplicationRequest` interfész: minden szerkeszthető mező + `rowVersion: string` (optimistic concurrency — a GET response-ból kapott RowVersion értéket vissza kell küldeni; 409 Conflict ha közben módosult)

### Reactive Form
- [ ] Ugyanazok a mezők, mint a CreateForm-ban
- [ ] Értékek betöltése a meglévő ApplicationDto-ból
- [ ] Mentés: `updateApplication()` hívás, siker → toast, nézet frissítés

### Authorization
- [ ] „Szerkesztés" gomb: `*hasRole="['Admin', 'Elnok', 'PalyazatiMunkatars']"`
- [ ] LOCKED állapotban a gomb disabled

## UX / Behaviour Specification

| Scenario | Expected behaviour |
|---|---|
| LOCKED pályázat | „Szerkesztés" gomb disabled, tooltip: „Lezárt pályázat nem módosítható." |
| Sikeres mentés | Toast: „Pályázat adatai frissítve." |
| Konkurencia konfliktus (409) | Toast: „A pályázat közben módosult. Töltsd újra az oldalt." |
