## Story

> As a **elnök**, I want to approve or reject the submission step.

---

## Scope

Approval/rejection panel visible only to Elnök/Admin on the Submission step. Shared `ApprovalPanelComponent` reused across US-021, US-052, US-071.

---

## Implementation Checklist

### Shared Component
- [ ] `src/app/shared/components/approval-panel/approval-panel.component.ts` + `.html` + `.scss`
  - „Jóváhagyás" zöld gomb, „Visszautasítás" piros gomb
  - Visszautasítás esetén: inline textarea a megjegyzésnek (kötelező)
  - `*hasRole="['Admin', 'Elnok']"`
  - `@Input() approveSuccessMessage: string` — toast szöveg jóváhagyáskor (pl. „Beadás jóváhagyva.")
  - `@Input() rejectSuccessMessage: string` — toast szöveg visszautasításkor (pl. „Beadás visszautasítva.")

### Service
- [ ] `WorkflowService.approveStep(appId: string, stepType: string, data: ApproveStepRequest): Observable<WorkflowStepDetailDto>` — POST `/api/v1/applications/:id/workflow/${stepType}/approve` (generikus; settlement esetén l. us-071-FE-1)
- [ ] `ApproveStepRequest` TypeScript interfész:
  ```ts
  interface ApproveStepRequest {
    isApproved: boolean;
    rejectionNote?: string; // required when isApproved=false
  }
  ```

### State / Data Flow
- [ ] Jóváhagyás után: step status frissítése UI-ban, toast „Beadás jóváhagyva."
- [ ] Visszautasítás után: toast „Beadás visszautasítva.", megjegyzés megjelenik a lépés nézetén
