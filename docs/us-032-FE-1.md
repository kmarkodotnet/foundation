## Story

> As an **adminisztrátor**, I want to correct a mistakenly recorded result so that the application reflects the actual decision.

---

## Scope

Admin-only "Correct Result" button on the Result step that reopens the result form pre-populated with existing data. Warns when switching from Won to Lost (cascading step impact).

---

## Implementation Checklist

### Feature Module
- [ ] `StepResultComponent` — add „Eredmény korrekciója" button (`*hasRole="['Admin']"`)
  - Pre-fills form with existing result data
  - Won→Lost transition: megerősítő dialog: „A módosítás lezárja a folyamat [4]–[9] lépéseit. Biztosan folytatod?"
  - Lost→Won transition: no special dialog
  - „Mentés" gomb → PUT /workflow/result/correct

### Service
- [ ] `WorkflowService.correctResult(appId: string, data: CorrectResultRequest): Observable<ApplicationDto>` — PUT /api/v1/applications/:id/workflow/result/correct
- [ ] `CorrectResultRequest` = azonos mezők mint `RecordResultRequest` (`isWon`, `awardedAmount?`, `resultDate?`, `resultIdentifier?`); külön típusként definiálandó a szándék egyértelművé tételéhez, bár a BE ugyanazt a DTO-t várja

### Authorization
- [ ] „Eredmény korrekciója" gomb: `*hasRole="['Admin']"` only

### State / Data Flow
- [ ] Sikeres korrekció: Toast „Eredmény sikeresen javítva.", lépések frissülnek
- [ ] Won→Lost után: lépések [4]–[9] szürkítve, NOT_APPLICABLE
- [ ] Lost→Won után: lépések [4]–[9] aktiválódnak
