## Story

> As a **pályázati munkatárs**, I want to create a budget plan with items so that we clearly define what we spend the awarded amount on.

---

## Scope

Creates or updates the BudgetPlan aggregate for an application, managing BudgetItems (CRUD). BudgetPlan belongs to the BudgetPlan workflow step. Provides summary comparison against awarded amount.

## API Contract

```
PUT /api/v1/applications/{id}/budget-plan
```

### Request Body
```json
{
  "notes": "Éves rendezvény és eszközbeszerzés",
  "items": [
    { "id": null, "name": "Nyári tábor", "type": "Event", "plannedAmount": 500000, "description": null, "sortOrder": 1 },
    { "id": "uuid", "name": "Laptop", "type": "Asset", "plannedAmount": 350000, "description": "Dell XPS", "sortOrder": 2 }
  ]
}
```

### Response: `BudgetPlanDto`
```json
{
  "id": "uuid",
  "applicationId": "uuid",
  "totalPlanned": 850000,
  "awardedAmount": 1000000,
  "difference": 150000,
  "notes": "...",
  "items": [...]
}
```

| Response | Description |
|---|---|
| **200** | BudgetPlanDto |
| **422** | Application not WON or step not active |

## FluentValidation Rules

| Field | Rule | Error message |
|---|---|---|
| `Items` | `NotEmpty()` to complete step | „Legalább 1 tétel szükséges a lépés lezárásához." |
| `PlannedAmount` | `GreaterThan(0)` | „A tervezett összegnek pozitívnak kell lennie." |
| `Name` | `NotEmpty()` | „A tétel neve kötelező." |
| `Type` | Valid enum value | „Érvénytelen tétel típus." |
