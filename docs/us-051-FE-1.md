## Story

> As a **pályázati munkatárs**, I want to edit or delete budget items so that the plan stays up to date.

---

## Scope

Edit and delete actions on each BudgetItem row in `StepBudgetPlanComponent`. Inline editing via expansion panel. Delete confirmation dialog with error handling for linked records.

---

## Implementation Checklist

### Feature Module
- [ ] `StepBudgetPlanComponent` — tétel soronként:
  - Szerkesztés ikon → `BudgetItemFormComponent` inline megnyitás, előre kitöltve
  - Törlés ikon → ConfirmDialogComponent
  - Törlés hiba kezelés: 422 → error toast „A tétel nem törölhető, mert kapcsolt számla/szerződés létezik."

### Service
- [ ] `BudgetPlanService.updateBudgetItem(appId: string, itemId: string, data: BudgetItemRequest): Observable<BudgetItemDto>` — PUT /api/v1/applications/:id/budget-plan/items/:itemId
- [ ] `BudgetPlanService.deleteBudgetItem(appId: string, itemId: string): Observable<void>` — DELETE /api/v1/applications/:id/budget-plan/items/:itemId

### Authorization
- [ ] Szerkesztés/törlés: `*hasRole="['Admin', 'PalyazatiMunkatars']"`

### State / Data Flow
- [ ] Szerkesztés után: helyi lista frissítése (nem API újrahívás) + összesítő újraszámítás
- [ ] Törlés után: toast „Tétel törölve.", elem eltávolítása a lokális tömbből (nem API újrahívás) + összesítő frissítés
- [ ] 422 hiba: error toast megjelenik
