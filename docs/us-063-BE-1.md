## Story

> As a **pénzügyes or elnök**, I want to see the full financial summary of the grant at a glance so that I know how much has been paid, how much remains, and where there are discrepancies.

---

## Scope

Financial summary is part of the `GET /invoices` response (already included via `InvoiceSummaryDto`). This story ensures the query returns all required summary fields and supports filtering/sorting.

---

## Implementation Checklist

### Application Layer
- [ ] `GetInvoicesQuery` (extend from US-060):
  - Filter: `IsPaid` (bool?), `DateFrom` (DateOnly?), `DateTo` (DateOnly?)
  - Sort: `SortBy` (issueDate | amount), `SortDirection`
- [ ] `InvoiceListDto`:
  - `Summary: InvoiceSummaryDto`
  - `Items: InvoiceDto[]`
- [ ] `InvoiceSummaryDto`:
  - `AwardedAmount`: from Application
  - `TotalPlanned`: from BudgetPlan
  - `TotalInvoiced`: sum of Invoice.Amount
  - `TotalPaid`: sum where IsPaid=true
  - `TotalUnpaid`: TotalInvoiced − TotalPaid
  - `Balance`: AwardedAmount − TotalInvoiced

### API Layer
- [ ] `GET /api/v1/applications/{id}/invoices?isPaid=&sortBy=&sortDirection=` — `[Authorize]`
- [ ] Returns `InvoiceListDto`

### Authorization
- [ ] All authenticated users

---

## API Contract

```
GET /api/v1/applications/{id}/invoices?isPaid=false&sortBy=amount&sortDirection=desc
```

### Response
```json
{
  "summary": {
    "awardedAmount": 1000000,
    "totalPlanned": 850000,
    "totalInvoiced": 400000,
    "totalPaid": 250000,
    "totalUnpaid": 150000,
    "balance": 600000
  },
  "items": [...]
}
```
