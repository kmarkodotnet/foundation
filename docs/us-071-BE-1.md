## Story

> As an **elnök**, I want to approve the settlement and close the application so that the process is officially complete and data is locked.

---

## Scope

Reuses `ApproveStepCommand` for the Settlement step type. On approval: sets application status to ClosedWon and locks all steps. Rejection keeps the step Active with a note.

---

## Implementation Checklist

### Domain Layer
- [ ] `WorkflowService.CloseApplicationWon(Application application)` — sets status=ClosedWon, all steps → LOCKED

### Application Layer
- [ ] `ApproveStepCommand` (existing) — Settlement path:
  - IsApproved=true:
    - Settlement step → Completed
    - Calls `WorkflowService.CloseApplicationWon()`
  - IsApproved=false:
    - Settlement step → Active, stores RejectionNote
- [ ] Validator: same as US-021 (RejectionNote required if !IsApproved)

### API Layer
- [ ] `POST /api/v1/applications/{id}/workflow/settlement/approve`
  - `[RequireRole(UserRole.Admin, UserRole.Elnok)]`
  - Returns `200 OK` with `ApplicationDto` (full state after close — intentionally differs from Submission/BudgetPlan approve which return `WorkflowStepDetailDto`; here the entire application state changes to ClosedWon and all steps are locked, so returning the full ApplicationDto is the natural response)

### Authorization
- [ ] Elnok, Admin

---

## API Contract

```
POST /api/v1/applications/{id}/workflow/settlement/approve
```

### Request Body
```json
{ "isApproved": true, "rejectionNote": null }
```

| Response | Description |
|---|---|
| **200** | ApplicationDto with CLOSED_WON status, all steps LOCKED |
| **403** | Not Elnok/Admin |
| **422** | Settlement step not in approvable state |

## FluentValidation Rules

| Field | Rule | Error message |
