## Story

> As an **elnök**, I want to approve the settlement and close the application so that the process is officially complete.

---

## Scope

Reuses `ApprovalPanelComponent` on the Settlement step. Approval triggers a special confirmation dialog warning that the application will be locked permanently.

---

## Implementation Checklist

### Feature Module
- [ ] `StepSettlementComponent` — extend with:
  - `<app-approval-panel approveSuccessMessage="Pályázat sikeresen lezárva. CLOSED_WON státusz." rejectSuccessMessage="Elszámolás visszautasítva.">`
  - Jóváhagyás gomb → speciális ConfirmDialog: „Biztosan lezárod a pályázatot? Ez után módosítás nem lehetséges."
  - Jóváhagyás után: CLOSED_WON badge, összes lépés LOCKED-ként jelenik meg, szerkesztés disabled

### Service
- [ ] `WorkflowService.approveSettlement(appId: string, data: ApproveStepRequest): Observable<ApplicationDto>` — POST /api/v1/applications/:id/workflow/settlement/approve
  - Visszatérési típus `ApplicationDto` (nem `WorkflowStepDetailDto`) — az elszámolás jóváhagyása lezárja a pályázatot, friss `ApplicationDto`-t ad vissza
  - Az általános `approveStep()` `WorkflowStepDetailDto`-t ad vissza; settlement esetén mindig `approveSettlement()` hívandó

### Authorization
- [ ] `ApprovalPanelComponent`: `*hasRole="['Admin', 'Elnok']"`

### State / Data Flow
- [ ] Jóváhagyás után: Router navigáció vagy UI frissítés → összes lépés locked, szerkesztés disabled
- [ ] Toast „Pályázat sikeresen lezárva. CLOSED_WON státusz."
- [ ] Visszautasítás után: megjegyzés megjelenik, Settlement lépés ACTIVE marad
