## Story

> As a **pályázati munkatárs**, I want to upload documents to a workflow step so that all paperwork is in one place.

---

## Scope

Shared `DocumentUploadComponent` reused across all workflow step views. File type and size validated client-side before upload. Progress bar during upload.

---

## Implementation Checklist

### Shared Component
- [ ] `src/app/shared/components/document-upload/document-upload.component.ts` + `.html` + `.scss`
  - „Dokumentum hozzáadása" gomb / drag-drop zone
  - Dokumentum típusa (MatSelect, CodeList alapján, required)
  - Megjelenítési név (MatInput, optional)
  - Fájl kiválasztás (accepted: .pdf,.docx,.xlsx,.jpg,.png,.tiff,.msg,.eml, max 50 MB)

  - `@Input() appId: string`

  - `@Output() uploaded: EventEmitter<DocumentDto>`

- [ ] `DocumentListComponent` — dokumentum lista (kártyák):
  - Fájlnév, típus, méret, feltöltötte, dátum
  - Letöltés / Előnézet gombok (US-081 scope)
  - Archivált dokumentumok szürkítve (szűrhető)

### Service

- [ ] `DocumentService.getDocuments(appId: string, stepId?: string): Observable<DocumentDto[]>` — GET /api/v1/applications/:id/documents

### Authorization
- [ ] „Dokumentum hozzáadása": `*hasRole="['Admin', 'PalyazatiMunkatars']"`

## UX / Behaviour Specification

| Scenario | Expected behaviour |
|---|---|
| Nem megengedett formátum | Azonnali kliens-oldali hiba, feltöltés nem indul |

| Feltöltés közben | Progress bar megjelenik |
| Sikeres feltöltés | Toast, dokumentum azonnal megjelenik |
