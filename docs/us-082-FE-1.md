## Story

> As a **pályázati munkatárs**, I want to upload a new version of a document while preserving previous versions.

---

## Scope

"Upload new version" action in `DocumentListComponent`. Version history panel accessible from document row.

---

## Implementation Checklist

### Feature Module
- [ ] `DocumentListComponent` — dokumentumonként (Admin/PalyazatiMunkatars):
  - „Új verzió feltöltése" gomb/ikon
  - Megnyíló `DocumentUploadComponent` (verziós módban, parent doc rögzített)
  - Verziótörténet toggle/expansion panel:
    - Verzió szám, dátum, feltöltötte
    - Letöltés ikon archív verzióhoz is

### Service
- [ ] `DocumentService.uploadVersion(appId: string, documentId: string, data: FormData): Observable<DocumentDto>` — POST /api/v1/applications/:id/documents/:documentId/versions
- [ ] `DocumentService.getVersions(appId: string, documentId: string): Observable<DocumentVersionDto[]>` — GET /api/v1/applications/:id/documents/:documentId/versions

### Authorization
- [ ] „Új verzió feltöltése": `*hasRole="['Admin', 'PalyazatiMunkatars']"`

### State / Data Flow
- [ ] Új verzió feltöltése után: lista lokálisan frissül (nem API újrahívás) — a feltöltés válasza az új `DocumentDto`, ezzel cserélni kell a listában a régit; új verzió aktívként jelenik meg
- [ ] Előző verzió archiváltként látható a verziótörténetben
- [ ] Toast „Új verzió sikeresen feltöltve."
