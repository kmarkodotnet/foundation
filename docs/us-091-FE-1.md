## Story

> As a **pályázati munkatárs**, I want to upload an .eml or .msg file and preview its content.

---

## Scope

Extends `EmailFormComponent` with optional .eml/.msg upload. "Preview" button on email records with attached files. Structured preview dialog.

---

## Implementation Checklist

### Feature Module
- [ ] `EmailFormComponent` — extend with fájl feltöltés:
  - .eml / .msg fájl input (optional, accept=".eml,.msg")
  - Client-side típus validáció

- [ ] `EmailRecordComponent` — kártyákon ha `hasAttachment=true`:
  - „Előnézet" gomb → `EmlPreviewDialogComponent`
  - „Letöltés" gomb → blob download

- [ ] `EmlPreviewDialogComponent` (MatDialog):
  - Feladó, Tárgy, Dátum, Törzsszöveg strukturáltan
  - „Bezárás" gomb

### Service
- [ ] `EmailService.attachFile(appId: string, emailId: string, data: FormData): Observable<EmailRecordDto>` — POST /api/v1/applications/:id/emails/:emailId/attachment
- [ ] `EmailService.getPreview(appId: string, emailId: string): Observable<EmlPreviewDto>` — GET /api/v1/applications/:id/emails/:emailId/preview
- [ ] `EmailService.downloadAttachment(appId: string, emailId: string): Observable<Blob>` — GET /api/v1/applications/:id/emails/:emailId/download

---

## API Consumed

| Method | URL | Body | Response |
|---|---|---|---|
| `POST` | `/api/v1/applications/:id/emails/:emailId/attachment` | `FormData` | `EmailRecordDto` |
| `GET` | `/api/v1/applications/:id/emails/:emailId/preview` | — | `EmlPreviewDto` |
| `GET` | `/api/v1/applications/:id/emails/:emailId/download` | — | Blob |

## UX / Behaviour Specification

| Scenario | Expected behaviour |
|---|---|
| Nem .eml/.msg feltöltés | Kliens-oldali hiba, feltöltés nem indul |
| „Előnézet" kattintás | EmlPreviewDialog megnyílik feladó/tárgy/dátum/törzs adatokkal |
| „Letöltés" kattintás | Blob letöltés |
