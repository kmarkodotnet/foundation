## Story

> As a **pályázati munkatárs**, I want to delete a mistakenly recorded email so that inaccurate records don't burden the communication history.

---

## Scope

Soft-deletes EmailRecord (and marks attached file for deletion). Only the creator or Admin can delete.

---

## Implementation Checklist

### Application Layer
- [ ] Command: `Application/Emails/Commands/DeleteEmailRecord/DeleteEmailRecordCommand.cs` — Guid ApplicationId, Guid EmailRecordId; implements `IApplicationCommand`
- [ ] Handler:
  - Authorization check: CurrentUserId == EmailRecord.CreatedByUserId OR role==Admin
  - Soft delete (IsDeleted=true)
  - If attachment exists: soft delete file record (physical file kept)

### API Layer
- [ ] `DELETE /api/v1/applications/{id}/emails/{emailId}`
  - `[Authorize]` (owner or Admin check in handler)
  - Returns `204 No Content`

### Authorization
- [ ] Creator or Admin (enforced in handler, not just role attribute)

---

## API Contract

```
DELETE /api/v1/applications/{id}/emails/{emailId}
```

| Response | Description |
|---|---|
| **204** | Deleted |
| **403** | Not creator or Admin |
