## Story

> As an **adminisztrátor**, I want to deactivate a granter so that inactive granters don't clutter the selection lists.

---

## Scope

Sets Granter.IsActive=false. Blocked if linked to any active applications. Inactive granters excluded from default autocomplete. Existing applications retain the granter name with "(inaktív)" marking.

---

## Implementation Checklist

### Application Layer
- [ ] Command: `Granters/Commands/DeactivateGranter/DeactivateGranterCommand.cs` — Guid GranterId
- [ ] Handler:
  - Checks: no active (non-archived, non-closed) applications reference this GranterId → if found, only warn (non-blocking); sets IsActive=false
  - Actually per AC: linked active applications → warning but not blocked; set IsActive=false
- [ ] `GetGrantersQuery` — `includeInactive=false` default; `isActive` filter on query

### API Layer
- [ ] `PATCH /api/v1/granters/{id}/deactivate` — `[RequireRole(UserRole.Admin)]`
  - Returns `200 OK` with `GranterDto`
- [ ] `PATCH /api/v1/granters/{id}/activate` — reactivate — `[RequireRole(UserRole.Admin)]`
  - Returns `200 OK` with `GranterDto`

### Authorization
- [ ] Admin only

---

## API Contract

```
PATCH /api/v1/granters/{id}/deactivate
```

| Response | Description |
|---|---|
| **200** | GranterDto with IsActive=false; includes `hasActiveApplications: bool` warning flag |
| **403** | Not Admin |
