## Story

> As a **pályázati munkatárs**, I want to receive a notification when a grant's submission deadline is approaching so that we can prepare and submit on time.

---

## Scope

Hangfire daily background job checks all active applications with upcoming submission deadlines. Sends in-app Notification records and emails. Configurable alert days from SystemSettings.

---

## Implementation Checklist

### Domain Layer
- [ ] `Notification` entity — UserId, Type (enum), Title, Body, EntityId (Guid?), EntityType?, IsRead (bool), CreatedAt
- [ ] `NotificationType` enum: DeadlineSubmission, DeadlineSpending, ApprovalRequest, ResultRecorded, etc.

### Application Layer
- [ ] Hangfire Job: `Notifications/Jobs/DeadlineAlertJob.cs`
  - Runs daily at 08:00
  - Queries: applications WHERE SubmissionDeadline = today + 7 days AND Status in (DRAFT, IN_PROGRESS)
  - Queries: applications WHERE SubmissionDeadline = yesterday AND Status in (DRAFT, IN_PROGRESS) [missed deadline]
  - For each: creates Notification records for relevant users (PalyazatiMunkatars, Elnok)
  - For each: sends email via `IEmailSender` (if user's NotificationPreferences.SubmissionDeadlineEmail=true)
- [ ] `INotificationService.CreateAsync(Guid userId, NotificationType type, string title, string body, Guid? entityId)`
- [ ] Registration in Hangfire scheduler

### Infrastructure Layer
- [ ] `EmailSender` (MailKit implementation)
- [ ] Hangfire DI registration
- [ ] SystemSettings: `DeadlineAlertDays` (default 7)

### API Layer
- No new endpoint (job-triggered)
