## Story

> As any **bejelentkezett felhasználó**, I want to view all my notifications in one place and mark them as read.

---

## Scope

Notification list endpoint for the current user. Mark single or all notifications as read. SignalR push for real-time delivery.

---

## Implementation Checklist

### Application Layer
- [ ] Query: `Notifications/Queries/GetMyNotificationsQuery.cs` — UserId (from ICurrentUserService), unreadOnly bool?
- [ ] Handler: returns notifications ordered by CreatedAt desc
- [ ] Command: `Notifications/Commands/MarkNotificationRead/MarkNotificationReadCommand.cs` — Guid NotificationId
- [ ] Command: `Notifications/Commands/MarkAllNotificationsRead/MarkAllNotificationsReadCommand.cs`
- [ ] `NotificationDto`: Id, Type, Title, Body, EntityId?, EntityType?, IsRead, CreatedAt

### Infrastructure Layer
- [ ] SignalR Hub: `NotificationHub.cs`
  - `[Authorize]` on hub
  - Push implementáció: `Clients.User(userId).SendAsync("ReceiveNotification", notificationDto)` — a kliens-oldali event neve kötelezően `"ReceiveNotification"` (a FE `NotificationBellComponent` erre iratkozik fel)
- [ ] `INotificationService.CreateAsync()` pushes via SignalR after DB insert

### API Layer
- [ ] `GET /api/v1/notifications?unreadOnly=` — `[Authorize]`
  - Returns `NotificationDto[]`
- [ ] `PATCH /api/v1/notifications/{id}/read` — `[Authorize]`
  - Returns `204 No Content`
- [ ] `PATCH /api/v1/notifications/read-all` — `[Authorize]`
  - Returns `204 No Content`
- [ ] SignalR endpoint: `/hubs/notifications`

### Authorization
- [ ] Current user's notifications only

## API Contract

```
GET /api/v1/notifications?unreadOnly=true
```

```json
[
  {
    "id": "uuid",
    "type": "DeadlineSubmission",
    "title": "Beadási határidő közeleg",
    "body": "Nyári tábor 2025 – határidő: 2025-10-01",
    "entityId": "uuid",
    "entityType": "Application",
    "isRead": false,
    "createdAt": "2025-09-24T08:00:00Z"
  }
]
```
