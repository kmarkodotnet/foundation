## Story

> As an **adminisztrátor**, I want to assign a role to a user so that they work with permissions matching their position.

---

## Scope

Updates AppUser.Role. Guards: at least 1 Admin must remain. Change takes effect on next JWT refresh.

---

## Implementation Checklist

### Application Layer
- [ ] Command: `Users/Commands/AssignRole/AssignRoleCommand.cs` — Guid UserId, UserRole NewRole
- [ ] Handler:
  - If removing Admin: checks at least 1 other active Admin exists → else 422
  - Updates AppUser.Role
- [ ] Validator: NewRole valid enum value

### API Layer
- [ ] `PUT /api/v1/users/{id}/role` — `[RequireRole(Admin)]`
  - Body: `{ "role": "PalyazatiMunkatars" }`
  - Returns `200 OK` with `UserListItemDto`

### Authorization
- [ ] Admin only

---

## API Contract

```
PUT /api/v1/users/{id}/role
```
```json
{ "role": "PalyazatiMunkatars" }
```

| Response | Description |
|---|---|
| **200** | Updated UserListItemDto |
| **422** | Would remove last Admin |
