## Story

> As an **adminisztrátor**, I want to deactivate or reactivate users without deleting their data.

---

## Scope

Sets AppUser.IsActive. Deactivated users blocked from login via Google auth callback. Admin cannot deactivate their own account.

---

## Implementation Checklist

### Application Layer
- [ ] Command: `Users/Commands/DeactivateUser/DeactivateUserCommand.cs` — Guid UserId
- [ ] Handler:
  - Guards: CurrentUserId != UserId (cannot self-deactivate)
  - Sets IsActive=false
- [ ] Command: `Users/Commands/ActivateUser/ActivateUserCommand.cs` — Guid UserId
- [ ] Handler: sets IsActive=true
- [ ] `GoogleLoginCommandHandler` (from US-001): checks IsActive → 403 if inactive

### API Layer
- [ ] `PATCH /api/v1/users/{id}/deactivate` — `[RequireRole(Admin)]`
  - Returns `200 OK` with `UserListItemDto`
- [ ] `PATCH /api/v1/users/{id}/activate` — `[RequireRole(Admin)]`
  - Returns `200 OK` with `UserListItemDto`

### Authorization
- [ ] Admin only; cannot target self

---

## API Contract

```
PATCH /api/v1/users/{id}/deactivate
PATCH /api/v1/users/{id}/activate
```
No body. Returns `UserListItemDto`.

| Response | Description |
|---|---|
| **200** | Updated UserListItemDto |
| **422** | Cannot deactivate own account |
