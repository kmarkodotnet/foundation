## Story

> As an **adminisztrátor**, I want to modify system settings so that the system behaves according to the foundation's needs.

---

## Scope

SystemSettings entity with configurable values. Settings take effect immediately. Used by deadline alert job and file upload validation.

---

## Implementation Checklist

### Domain Layer
- [ ] `SystemSettings` entity (singleton, Id=1) — DeadlineAlertDays (int, default 7), SpendingDeadlineAlertDays (int, default 14), MaxFileSizeMB (int, default 50), OrganizationName (string), DefaultUserRole (UserRole, default Megtekinto)

### Application Layer
- [ ] Query: `Settings/Queries/GetSystemSettingsQuery.cs` — returns `SystemSettingsDto`
- [ ] Command: `Settings/Commands/UpdateSystemSettings/UpdateSystemSettingsCommand.cs` — all settings fields
- [ ] Handler: updates singleton SystemSettings
- [ ] Validator: DeadlineAlertDays > 0, MaxFileSizeMB > 0, MaxFileSizeMB ≤ 200

### Infrastructure Layer
- [ ] EF Core config: `SystemSettingsConfiguration.cs`
- [ ] Migration: `AddSystemSettingsTable` + seed row (Id=1, defaults)
- [ ] `ISystemSettingsProvider` singleton service: caches settings, invalidated on update

### API Layer
- [ ] `SettingsController`:
  - `GET /api/v1/settings` — `[RequireRole(Admin)]`
  - `PUT /api/v1/settings` — `[RequireRole(Admin)]`
  - Returns `SystemSettingsDto`

### Authorization
- [ ] Admin only

---

## API Contract

```
GET /api/v1/settings
PUT /api/v1/settings
```

### Request/Response Body
```json
{
  "deadlineAlertDays": 7,
  "spendingDeadlineAlertDays": 14,
  "maxFileSizeMB": 50,
  "organizationName": "Minta Alapítvány",
  "defaultUserRole": "Megtekinto"
}
```

| Response | Description |
|---|---|
| **200** | SystemSettingsDto |
| **422** | Invalid values (negative numbers, etc.) |

---

## FluentValidation Rules

| Field | Rule | Error message |
|---|---|---|
| `DeadlineAlertDays` | `GreaterThan(0)` | „Az értéknek pozitívnak kell lennie." |
| `MaxFileSizeMB` | `InclusiveBetween(1, 200)` | „A fájlméret korlát 1–200 MB között lehet." |
